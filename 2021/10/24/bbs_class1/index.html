    • 调用父类的函数\构造方法

  1. 重写：作用在子类

    • 对父类的方法函数进行重定义；
    • 条件：父类存在此方法，子类继承此方法、子类声明并重定义此方法；
    • 重写目的：由于父类的方法并不满足所需功能，因此需要对此方法进行重写改编至满足需求；子类对象存在此方法的重写定义，就不会再调用父类的方法；

    “==”与equals的区别：

    “==”作用在基本类型（其中不包括String），比较的是数值，作用在其他类型比较的是地址；

    “equals” 原本是在Object类中同“==”，而在String类内将其由父类Object(所有子类的默认父类)中继承下来的equals函数进行了重写；增加了其的用法；

八、接口

  • 概念

    一个标准、一个协议

    内存、主板；

  • 接口的语法

  • 接口的作用

    • 提高效率
    • 需求变化，实现可以任意变更，只要接口一样
    • 实现多态：抽象类、接口

  • 抽象类和接口的区别

    接口是用来被实现的；

    抽象类是用来被继承的；所体现的是（is a …）一种类别；

    1. 语法层面：

      • ``` java
        //抽象类
        abstract class 类名{

        }

    2. 编程层面

      • java不能多继承，但是可以实现多个接口
      • 抽象类体现继承关系，可以实现默认行为—抽象类中定义的具体函数；抽象类也有抽象函数；

    3. 设计理念

      • 抽象类体现的是继承关系，is a,反映出子类和父类本质上是一致的
      • 接口体现的是实现关系，并没有要求实现类在本质上一致；like a

  13. 基本数据类型和引用数据类型在内存上的区别：

    • 类、接口和数组为引用数据类型；引用数据类型在堆里，基本数据类型在栈里，栈空间小且连续，往往放在缓存中，适合存放基本数据类型。

    • int[]类型属于引用数据类型，在内存中变量本身存储在栈中，值存储在堆中，栈中存储的是地址，指向值存储的堆中；

  14. 变量类型运算：

    • 整型：int、byte、long、short
    • 浮点型：float、double
    • 字符型：char
    • 布尔型：boolean
    • 数据类型间的转换：
      • 自动类型转换：（编译器自动完成类型转换）
        1. 具体转换规则：byte、short、char - -> int —> long —> float —> double
        2. byte\short\char属于同一层次类型，之间不会相互转换，运算时会首先转换成int型；
        3. 当多种数据类型进行混合运算时，java会首先将所有数据转换为表示范围最大的数据类型再进行运算；
      • 强制类型转换：（通过代码强制编译器完成类型转换）
        1. 转换规则：从存储范围大的类型转换为存储范围小的类型。
        2. 具体转换规则：double—>float—>long —>int —>byte\short]char
        3. 代码语法格式：（想要转换到的数据类型）需要转换的数值变量

    • 变量的默认初始值

      | 类型 | 默认初始值 |
      | ——————————————- | ————— |
      | 整数型（byte\short\int\long） | 0 |
      | 单精度浮点型(float) | 0.0f |
      | 双精度浮点型(double) | 0.0d |
      | 字符型(char) | \u0000 |
      | 布尔类型(boolean) | false |
      | 引用类型变量 | null |

  15. 类与封装：

    封装的目的：1. 简化编程；2. 增强安全性；

    为什么java编译文件中只能有一个public类？

    答：在java中，为使在多个编译单元中有一个统一的接口，即public class作为唯一接口，将所有的接口子类和包统一起来管理与实施；

    此外，在java中public类可以不存在，但存在必须唯一；

    public类的名称必须完全与含有该编译单元的文件名相同，包含大小写；

  16. str.equals(String);中str必须存在，否则会出现空指针异常；

  17. 对于字节流和字符流：

    • 字节流：InputStream/OutStream
    • 字符流：Reader/Writer；

  18. 如果对同一个线程对象两次调用start()方法，在运行时会抛出一个异常；

  19. Collection是java.util包下的接口，它是List和Set集合结构的父接口；

    • 优先级高的并不一定会马上执行；

    • sleep方法会阻塞一个线程并不会终止；

    • 创建一个新的线程时也不会终止另一个线程；
    • 当抛出一个异常后程序会结束，所以线程也会被终止。

  21. The different of both of Thread.start() and Thread.run() ::

    • run():是函数接口的实现；单独调用run，就是调用普通函数，不会收到线程之间的影响；该运行就会运行；
    • start():在线程运行时，会默认调用run()方法，但是线程的开始，需要分配的cpu资源。否则会处于堵塞状态。从而start()默认调用run()也无法正常运行；
  22. join()函数；线程调用join函数系统将会优先将此线程执行结束再继续执行原来的线程；
  23. 字符流与字节流的转换注意事项：
    • 输出端>>——————>>>>——————->>输入端 ：字节流和字符流的转换主要取决于输入输出端的需求；输入端为字节流，输出端需要字符流，则需要包装为字符流；

在小程序中必不可少的文件有：
app.js:里面会自动列出了项目加载等生命周期函数；也可以自动在其中添加全局可调用函数；
app.json:导航navigation（顶部）的配置信息，背景时dark还是light；navigationBartext文本显示等
app.wxss:全局样式文件小程序的全体布局情况，如，字体等主题的配置；
project.config.json:这是微信小程序项目注册配置文件，其中包含了开发者信息，以及各种程序注册信息；
sitemap.json:新建项目时系统自带的，提示微信小程序开发者的帮助文档页面；

app.wxss

  • 全局样式配置文件(类似css)
  • stylename特指样式名称，其可运用方式为在标签内标注:’ class = “stylename” ‘;
  • 据我所了解，前css采用盒子模型（BoxModel）依赖于display+position+float方式实现；
  • 每一个元素即为一个盒子，而wxss(为这些盒子定义了一个样式)
  • 样式例举：
    • width、height：元素的高度和宽度；
    • padding(页内距):盒子外部和元素距离；
    • margin(外边距):盒子外部和周围元素的距离，其中参数有：(margin-bottom、margin-top、margin-left、margin-right)；
  • 盒子模型即为基础的样式封装；(默认使用W3C标准) 效果图：

    作用

    CSS作为描绘HTML的一种样式语言，它为HTML语言的提供不同的呈现方式；
    正如电影剧本，通过描述一个角色的特征外貌，导演和化妆师能准确的将这形象呈现在荧幕上；

    盒子模型

    盒子模型（BoxModel），其原理为将元素通过CSS进行包装，想想成盒子，有外层包装和内层包装，其中外层包装加内层包装决定了元素所占用位置。
    在CSS中为了能将样式统一化处理，将相同样式的元素定义为相同的一个类。操作方法即在

    盒子模型：运用外边距：margin+内边距：padding，对div进行修饰；

    • padding：内边距，元素距离box的距离
    • margin:外边距，盒子与其他元素的距离
    • position:定位方式：
      • absolute：绝对定位。以左上角为0px 0px；搭配left:200px;top:200px;进行使用；
      • relative:相对定位，相对原来的位置进行位置的移动，但原位置不可占用；

    flex弹性容器

    弹性布局（Flexbox layout）
    主要功能是让元素能更好的适应不同设备的屏幕显示；精确管理子元素；
    在盒子容器中，我们将样式想象成一个包装盒子，在弹性容器中，在子元素和父元素的加入之下。我们需要将一个模块的元素内部项目items进行有效的管理。
    一个比较大的模块，在经过有效的处理可以达到清晰明了的效果。因此我们引入flex弹性容器，对子元素进行统一管理。


    上述代码解释：
    flex-direction：此定义了子元素在容器内部的排列方式；row行排列也就是从左到右；column为列排列，从上到下；reverse为逆向排列；
    flex-wrap：此定义了子元素的排列情况反馈，如nowrap的情况下，元素遇到边界不会换行；wrap的情况下，弹性容器为多行状态，子项溢出会自动放置到新行；子项内部会断行；

    Tomcat的安装与配置

    Tomcat是Apache公司开发的免费的小型web服务器，用于管理web工程；提供jsp和Servlet的支持

    • 下载：

      Apache Tomcat® - Apache Tomcat 8 Software Downloads

      下载Zip文件可以解压缩直接使用；

    • 安装与配置

      1. 解压zip文件；

      2. 配置Tomcat环境

        • 新建系统环境变量：

          name:CATALINA_HOME

          path:localpath\apache-tomcat-8.5.78-windows-x64\apache-tomcat-8.5.78;

          注：localpath为解压缩的目录；

        • 添加变量至Path变量内：

          1. %CATALINA_HOME%\lib
          2. %CATALINA_HOME%\bin
          3. %CATALINA_HOME%\lib\servlet-api.jar

      3. 测试是否安装成功：

        1. 进入Tomcat的主目录下的\bin下

          • 运行目录下的startup.bat
          • 在该目录下运行cmd(命令行)使用命令”catalina run“命令进行运行；

        2. 查看是否服务器运行成功：

          在浏览器中进入：

          http://localhost:8080/

          • 在服务器正常运行情况下，能够进入Tomcat自带的测试页面；
          • 若运行不成功，有两种情况：
            1. 服务器尚未正常运行
            2. config文件中配置的端口有修改；

        3. 服务器未能运行成功的原因：

          JAVA_HOME未正确配置

          CATALINA_HOME未正确配置；

    • Tomcat文件及文件夹结构详情解析：

      • bin：专门用来存放Tomcat服务器的可执行程序

      • conf：专门用来存放Tomcat服务器的配置文件

    请求的HTTP协议格式

    GET请求头格式：

    1. GET请求

      • 请求行

        (1) 请求的方式 GET

        (2)请求的资源路径[+?+请求参数]

        (3)请求的协议和版本号

      • 请求头：键:值

        Accept：告诉服务器，客户端可以接收的数据类型

        Accept-Language: 告诉服务器可以接收的语言类型

        • zh_CN:中文中国

        • en_US:英文美国

        User-Agent:浏览器信息

        Accept-Encoding：告诉服务器，客户端可以接受的数据编码格式；

        Host：告诉服务器ip和端口号

        Connection:告诉服务器请求连接如何处理

        • Keep-Alive:回传数据不要马上关闭，保持一小段时间的连接；
        • Closed:马上关闭

    2. POST请求

      • 请求行

        (1) 请求的方式 GET

        (2)请求的资源路径[+?+请求参数]

        (3)请求的协议和版本号 HTTP

      • 请求头

        Accept：告诉服务器，客户端可以接收的数据类型

        Accept-Language: 告诉服务器可以接收的语言类型

        Referer:表示请求发起时，浏览器地址栏中的地址（从哪里来）

        Content-Type:表示发送的数据的类型

        • application/x-www-form-urlencoded

          ​ 表示提交的数据格式是：name=value&name=value,然后对其进行url编码

          ​ url编码是把非英文内容转换为：%xx%xx(16进制)

        • multipart/form-data 表示以多端的形式提交数据给服务器（以流的形式提交，用于上传）

        • Content-Length:表示发送的数据的长度

        • Cache-Control:表示如何控制缓存no-cache不缓存

关于win10下的cuda及pytorch的安装

​ 前言：cuda驱动时nvidia显卡的驱动；通过此驱动显卡的的算力作用在图像机器学习等神经网络的模型的训练；由于是nvidia显卡的驱动；因此需要注意的是，只有安装为nvidia显卡的电脑才能安装此驱动；

  • 安装cudatookit

    在安装cuda之前，首先需要确认电脑显卡驱动版本；在命令行通过nvidia-smi命令进行查看

    “Driver Version”,之后的安装完成cuda也是通过此命令进行查看

    • 一点小建议：新手安装cuda比较建议安装11.2左右的版本；这样即使出现问题也可以能够在网络上更好的找到解决方法；

    • 在官网下载好安装包

      此次步骤所下载安装的版本是11.4的：

      CUDA Toolkit 11.4 Downloads | NVIDIA Developer

    • 安装步骤：

      1. 推荐启动安装比较官方的程序时养成习惯使用管理员身份运行；这样程序安装的时候不会出现读写文件的问题出现；

      2. 接下来是进行安装路径的选择；其默认推荐的是C:/user/‘username’/AppData/Local/“目录下的文件夹中，其中我们可以看到许多程序默认安装都是安装在此文件夹中如“QQ\Wechat”等通讯工具，因此可以推断此处的文件读取权限不会影响程序的运行；（记住cuda的安装路径，默认“C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4”）

        （并不重要）补充说明：如果你的电脑只有一个硬盘而且还是固态硬盘，就不需要过于考虑什么“我装这么多程序在C盘中会不会导致C盘十分卡，改装在D盘中”，这种想法并不是正确的；当你电脑只有一个硬盘的时候，其实虚拟分区只是存储的地址划分，并不影响硬盘的读取速度；影响读取速度的只有硬盘自身同内存之间的通道读取速度；是由硬盘所决定的；常见的情况就是，大多数固态硬盘的读取速度相较于机械硬盘的读取速度较快；因此电脑的运行速度取决于硬件的影响；

      3. 使用默认推荐的安装地址，无脑下一步，直到选择安装选项，点开CUDA列表，取消掉其中的“visual studio integration”，这个对于开发者来说并不是必要得，首先这是作用于微软得VS编译软件的。在笔者看来VS过于臃肿，没有针对性，并不是很喜欢。且不利于新手操作，占用空间大。学习成本过高，是个次要选择；

      4. 之后就是无脑下一步的安装；

  • 安装cuDNN

    cuDNN是CUDA驱动所依赖深度神经网络的GPU加速的库文件，其强调性能、易用性和内存开销

    1. 下载cuDNN

      此次安装所使用的是cuDNN 11.4

      cuDNN Archive | NVIDIA Developer

    2. 解压缩，将解压缩后的文件复制到cuda的安装路径：“C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4”此时会弹出没有权限，默认继续将其中文件进行覆盖即可；

  • 查看系统环境变量是否添加好，其中有：

    • CUDA_PATH;C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4
    • CUDA_PATH_V11_4;C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4
    • PATH中新建：
      1. C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4\bin
      2. C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4\libnvvp
      3. C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4\include
      4. C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.4\lib

  • 使用命令行查看是否成功安装CUDA：

  • OVER(运行成功，结束)

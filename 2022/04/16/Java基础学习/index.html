第一章 初识java

  • JAVA语言于1995年由Sun公司推出；后由Oracle公司收购；

  • Java分有三个体系：

    • JavaSE:称为java的标准版，包含java基础类和语法。主要用于开发图形用户界面（GUI）、复杂逻辑和高性能的桌面应用程序。JavaSE分为四个组成部分：
      1. JVM(Java Virtual Mechine)java虚拟机；可以将java编译好的文件翻译为响应的机器语言；
      2. JDK(java Development kits)java语言软件开发工具，其中包括如javac等工具；JDK是java开发的核心，一般下载好的JDK里面包含了（JRE）JAVA的运行环境(JVM+java系统类库)和java工具
      3. JRE(Java Runtime Environment):Java运行环境
      4. java语言；
    • JavaEE：Java企业版，以javaSE为基础，定义一系列的服务，API，协议等，增加了企业级应用程序的类库。主要用于开发和部署分布式、基于组件、安全可靠、可伸缩和易于管理的企业级应用程序。
    • JavaME:称为Java微型版，主要用于开发消费性电子产品或嵌入式系统中的应用程序，开发具有有限的连接、内存和用户界面能力的设备应用程序。如手机等；

  • java程序的执行流程：

    1. .java->使用java语言编写出java源文件；
    2. .class->通过jDK中的javac对java源文件进行complie

    以上式编译过程；(编写)—>(编译)

    下面式加载过程：(加载)—>(验证)—>(准备)—>(解析)—>(初始化)

    1. 类装载器->通过一个类的全限定名来获取顶一次类的二进制字节流；
    2. 字节码校验器->验证确保Class文件的字节流中包含的信息符合当前虚拟机的要求，并且不会危害到虚拟机的安全；
    3. 解析器->解析阶段是虚拟机将class常量池内的符号应用替换为直接引用的工程
    4. 操作系统平台；

  • java语言的编译执行具有什么的特点：一次编译、处处运行；

  • java三种核心机制：

    1. jvm(Java Virtual Machine)Java虚拟机
    2. GC(Garbage Collection)垃圾收集机制
    3. CS(code Security)代码安全性检测
